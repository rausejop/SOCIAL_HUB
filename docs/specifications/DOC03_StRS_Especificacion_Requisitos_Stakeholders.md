# Especificación de Requisitos de Stakeholders (StRS) - SocialHub

## 1. Introducción
### 1.1 Propósito
Documentar formalmente las necesidades de los usuarios y operadores de SocialHub.

## 2. Requisitos del Sistema
### 2.1 Procesos del Sistema
- Flujo de creación de posts (max 280 caracteres).
- Visualización de Timeline infinito.
- Gestión de borradores (Drafts).
- Seguimiento de usuarios.

## 3. Requisitos del Usuario
### 3.1 Funcionalidad
- El usuario quiere poder interactuar en tiempo real.
- El usuario requiere acceso offline a posts cacheados.
- El usuario desea un sistema de búsqueda eficiente.

## 4. Concepto de Ciclo de Vida
### 4.1 Escenarios Operativos
- Publicación de pensamientos cortos.
- Consulta de tendencias (Trending).
- Notificaciones de interacción.
